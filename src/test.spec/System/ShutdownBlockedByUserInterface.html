What if the shut down must be blocked by the User Interface?

In some cases the User Interface must be able to block an unforced shut down process. For instance when the user has unsaved files open. In this case the user interface should be able to block the shut down process on the users request.

Example

When a shutdown is requested the system will start the shut down process. However if for instance the user has unsaved files open then the User Interface should be able to stop the shut down process so that the users files can be saved correctly. Once this is done the User Interface can re-start the shut down process again. If this process is not stopped then it should run succesfully.
